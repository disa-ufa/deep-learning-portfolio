# Краткий отчёт

## Цель

Создать нейро-сотрудника для фриланс-разработчика и отработать базовый RAG-подход: база знаний → векторное хранилище → найденный контекст → ответ модели.

## База знаний

Использовалась база знаний в Google Docs. Сначала она была представлена в плохо структурированном виде, затем структурирована по схеме:

```text
Заголовок 1 уровня
Заголовок 2 уровня
Фрагмент
```

Такая структура помогает поиску извлекать более релевантные фрагменты.

## Оптимизация

- добавлен подробный системный промпт с ролью нейро-сотрудника;
- задан строгий формат ответа;
- добавлено правило не выдумывать сроки и стоимость при нехватке данных;
- фрагменты базы знаний передаются в модель с H1/H2-контекстом;
- добавлена контрольная ячейка для проверки без Gradio.

## Итог

Нейро-сотрудник выполняет свои обязанности: анализирует заявку, задаёт уточняющие вопросы, предлагает решение, оценивает сроки/стоимость и пишет готовый ответ клиенту.
